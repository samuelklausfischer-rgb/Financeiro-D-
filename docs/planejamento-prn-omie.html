Hoje

Dashboard usa apenas dados locais de pagamentos e recebedores.

OMIE entra so em consulta pontual dentro do PRN.

Financeiro ainda depende de visao espalhada e planilhas paralelas.

Nao existe visao unificada de saldo, pagar, receber e extrato.

Parcial

Futuro

Dashboard alimentada diretamente pela OMIE em tempo real.

Visao de pagar, receber, extrato e caixa em uma unica tela.

Operacao financeira centralizada e mais rapida.

Conciliacao e auditoria entre PRN e OMIE automatizada.

Completo

2. O que ja existe no projeto

Integracao atual

O projeto ja possui uma integracao OMIE pontual usada para consultar pagamentos e vincular com itens do PRN.

Arquivos: src/services/prn-omie.ts e supabase/functions/prn-omie-lookup/index.ts

O que falta

A dashboard principal ainda nao e alimentada amplamente pela OMIE. Ela usa apenas dados locais de pagamentos, recebedores e alertas.

Arquivo: src/pages/dashboard/index.tsx

3. Desenho simples da arquitetura

Fluxo de como os dados da OMIE vao chegar ate a tela do financeiro:

OMIE

Dados oficiais do financeiro: pagar, receber, saldo, extrato, PIX, boletos.

→

Integracao Segura

Edge Function busca, trata e audita os dados antes de entregar ao site.

→

Banco Local

Dados organizados para leitura rapida e historico confiavel.

→

PRN Financas

Dashboard viva, telas operacionais e inteligencia financeira.

Regra importante: a OMIE nunca deve ser chamada diretamente do navegador com a chave de API. Toda comunicacao passa por uma camada servidora segura.

8. Roadmap de implantacao por etapas

1

Conexao segura com OMIE

Criar camada de integracao no servidor para consultar a OMIE com seguranca, logs e tratamento de erros. Nesta fase o site so le, nao escreve nada na OMIE.

Entrega: comunicacao funcionando, autenticacao validada, logs ativos.

2

Sincronizacao base

Trazer para o banco local: contas correntes, contas a pagar, contas a receber, extrato e resumo financeiro. Isso cria uma base rapida e historica para a dashboard.

Entrega: tabelas locais alimentadas, dados organizados, historico preservado.

3

Dashboard viva

Substituir os cards atuais por dados reais da OMIE: saldo, pagar, receber, atrasos, vencendo e previsto x realizado.

Entrega: painel executivo alimentado pela OMIE, atualizacao frequente.

4

Operacao diaria

Criar ou reforcar telas de contas a pagar, contas a receber, boletos, PIX e extrato por conta.

Entrega: central de trabalho para o financeiro operar melhor.

5

Auditoria PRN + OMIE

Cruzar dados da planilha PRN com titulos da OMIE. Gerar divergencias, alertas e recomendacoes de conciliacao.

Entrega: conferencia semi-automatica, reducao de retrabalho manual.

6

Inteligencia financeira

Alertas automaticos, fornecedores criticos, categorias fora do padrao, desvio de caixa e titulos sem conciliacao.

Entrega: sistema proativo que avisa antes do problema.

10. Regras para a integracao ser eficiente

O que fazer

1. Usar OMIE como fonte confiavel de financeiro.

2. Salvar no banco local apenas o necessario para leitura rapida.

3. Manter historico das sincronizacoes e logs de erro.

4. Separar dados operacionais de dados analiticos.

5. Comecar por leitura antes de automacao de escrita.

O que evitar

1. Chamar OMIE direto do navegador com chave de API.

2. Misturar payload bruto da OMIE direto na interface.

3. Fazer cada card consultar uma API diferente em tempo real.

4. Criar telas sem camada de normalizacao.

5. Transformar PRN e OMIE em fontes conflitantes sem regra de precedencia.

11. O que traz resultado mais rapido

Se precisar mostrar valor logo no comeco, estas sao as entregas com maior impacto no menor prazo:

1. Painel do dia

Usando /financas/resumo/, ja e possivel mostrar saldo, a pagar, a receber e atrasados em uma unica tela.

2. Contas a pagar reais

Usando /financas/contapagar/, o financeiro passa a ver titulos reais com vencimento e status atualizado.

3. Extrato rapido

Usando /financas/extrato/, da para mostrar saldo e ultimos movimentos por conta.
